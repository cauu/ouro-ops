## 一、文档说明

### 1.1 适用范围

本测试用例基于《详细设计文档 v1.0》编写，覆盖：

- Tauri IPC Commands（Pool / Machine / Deploy / Upgrade / KES / Monitor）
- Tauri 事件通道
- Sidecar JSON-RPC 协议
- 数据模型与迁移
- 业务流程状态机
- 错误码与恢复策略
- 前端路由与关键组件
- 端到端关键路径

### 1.2 用例标识与优先级

| 优先级 | 说明 |
| :--- | :--- |
| P0 | 核心功能，阻塞发布 |
| P1 | 重要功能，应在 MVP 内完成 |
| P2 | 增强/边界场景，可后续迭代 |

| 类型 | 说明 |
| :--- | :--- |
| 单元 | 单模块/单函数，可 Mock 依赖 |
| 集成 | 多模块/跨进程（Rust↔Sidecar、IPC） |

### TC-POOL-001 pool_init 首次初始化成功

| 项目 | 内容 |
| :--- | :--- |
| 优先级 | P0 |
| 类型 | 集成 |
| 前置条件 | 应用首次启动，`pool` 表为空 |
| 步骤 | 1. 调用 `pool_init({ ticker: "OURO", network: "preprod", margin: 0.02, fixed_cost: 340000000 })` |
| 预期结果 | 返回 `Pool` 对象，含 `id`、`ticker`、`network`、`margin`、`fixed_cost`、`created_at`；DB 中 `pool` 表有一条记录 |
